WhatsApp Business API

Configuración de la conexión con Meta Cloud API para enviar y recibir mensajes de WhatsApp.

Estado de conexión

Pendiente de configurar Meta Cloud API — Número dedicado Guadalajara

Para conectar WhatsApp necesitas obtener 3 datos desde developers.facebook.com:
1. Phone Number ID — ID del número registrado
2. WABA ID — WhatsApp Business Account ID
3. Access Token — Token de acceso (permanente desde Sistema > Usuarios del sistema)

Credenciales Meta

Lo encuentras en developers.facebook.com → Tu App → WhatsApp → Configuración
Genera un token permanente en Meta Business Suite → Sistema → Usuarios del sistema
Cualquier string secreto. Lo usarás al configurar el webhook en Meta.

Configuración del Webhook

Meta necesita un endpoint HTTPS para enviarte los mensajes entrantes. Usaremos una Supabase Edge Function como webhook.

Configura este URL en Meta App → WhatsApp → Configuración → Webhook
Pasos para activar el webhook:
  1. Ve a developers.facebook.com → Tu App
  2. WhatsApp → Configuración → Webhook
  3. Pega la URL de arriba y tu Verify Token
  4. Suscríbete a: messages
  5. Despliega la Edge Function (ver README)

Etapas del embudo

Pipeline actual

Arrastra para reordenar. Los nombres, el color y hasta cuántas etapas tengas son tuyos — cámbialos como los necesites.

Tip: de dónde salen estas 6 etapas

Son solo el punto de partida — las puedes renombrar, agregar o quitar arriba. Así es como las pensamos originalmente para Cremalleras LR:
Pendiente → llegó un WA sin atender
Contestado → ya respondiste, abriste conversación
Prospecto → tiene interés real, está calificado
Cotizado → enviaste precio/PDF
Agendado → acordaron fecha/entrega
Cerrado → venta realizada ✅

Automatizaciones

Disparadores automáticos para seguimiento de leads. Se construyen directo en código, sin depender de n8n ni de otro motor externo (más simple y sin costo extra de suscripción).

Estado

En diseño, todavía sin construir

Los ejemplos de abajo son ilustrativos de lo que se puede automatizar — todavía no están conectados a nada real, es la primera automatización pendiente de diseñar y construir.

Ejemplos de lo que se podría automatizar

Seguimiento usuario final — 1 hora
Si no hay respuesta en 60 min, enviar mensaje de seguimiento
Seguimiento mecánico — 20 min
Mecánicos reciben seguimiento más rápido (20 min sin respuesta)
Recordatorio de cita
1 hora antes de la cita agendada, enviar recordatorio
Mensaje fuera de horario
Si escriben fuera de horario (8pm-8am), respuesta automática

Regla de 24 horas (Meta)

Después de que un cliente escribe, tienes 24 horas para responder libremente.
Los seguimientos de 1h y 20min caen dentro de esa ventana ✅
Para follow-ups más tarde (quincena, etc.) necesitas plantillas pre-aprobadas por Meta.
